# Unstructured + Weaviate RAG App

A full RAG (Retrieval-Augmented Generation) pipeline built with Streamlit. Upload documents,
ask questions in plain English, and get grounded answers generated by Claude — backed by
hybrid vector + BM25 search over a local Weaviate instance.

## How it works

```
Document upload
     │
     ▼
Unstructured API  →  parse + chunk by section
     │
     ▼
OpenAI embeddings  →  text-embedding-3-small (1536 dims)
     │
     ▼
Weaviate (local)   →  hybrid index (vector + BM25)
     │
     ▼
Query → retrieve top 5 chunks → Claude claude-sonnet-4-6 → natural language answer
```

## Prerequisites

- Python 3.10+
- Docker

## Setup

### 1. Start local Weaviate

```bash
docker run -d -p 8080:8080 -p 50051:50051 --name weaviate \
  cr.weaviate.io/semitechnologies/weaviate:latest
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure API keys

```bash
cp .env.example .env
```

Edit `.env`:

```
UNSTRUCTURED_API_KEY=your_unstructured_key
OPENAI_API_KEY=your_openai_key
ANTHROPIC_API_KEY=your_anthropic_key
```

| Key | Where to get it |
|-----|----------------|
| `UNSTRUCTURED_API_KEY` | [unstructured.io](https://unstructured.io) |
| `OPENAI_API_KEY` | [platform.openai.com](https://platform.openai.com) |
| `ANTHROPIC_API_KEY` | [console.anthropic.com](https://console.anthropic.com) |

### 4. Run the app

```bash
streamlit run app.py
```

## Usage

**Ingesting documents**

1. Use the sidebar to upload one or more `.pdf`, `.docx`, `.txt`, or `.html` files
2. Click **Ingest** — each file is parsed, chunked, embedded, and indexed into Weaviate
3. Re-ingesting the same file is safe: existing chunks are replaced, not duplicated

**Asking questions**

1. Type a question in the search box (e.g. "Tell me about Twinly" or "What are the key skills?")
2. Adjust the **Alpha** slider to tune the search blend:
   - `0` = pure BM25 keyword matching
   - `1` = pure vector semantic search
   - `0.5` = balanced hybrid (default)
3. Claude reads the retrieved chunks and generates a grounded answer
4. Expand the **Sources** section to see which chunks were used
